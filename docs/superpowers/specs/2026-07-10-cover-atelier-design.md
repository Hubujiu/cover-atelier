# Cover Atelier 设计说明

## 目标

构建一个完全离线运行的博客文章封面编辑器。用户可以上传本地背景图，在画面中央叠加带高斯模糊效果的半透明玻璃横条，编辑标题和副标题，选择或上传字体，并将最终封面导出为 PNG。

## 已确认范围

- 首版支持：上传背景图、编辑标题和副标题、选择字体、调整玻璃横条、导出 PNG。
- 默认输出尺寸：1600 × 900，比例 16:9。
- 所有处理都在浏览器本地完成，不使用后端、云存储、CDN、远程字体服务或第三方接口。
- 首版不包含：账号系统、模板云端保存、历史记录、多图层编辑、在线协作。

## 设计方向

这是一个面向博客作者的封面工作台，而不是通用后台。视觉语言是“暗色编辑工作台 + 明亮相片画布”：工作区使用深蓝黑色，封面画布拥有独立的阴影和细边框，玻璃标题牌延续参考图的柔和、安静、轻杂志感。

设计参数：

- 画布外层：深墨蓝背景、低对比边界、宽松留白。
- 封面：16:9 比例，图片使用 cover 裁剪，焦点保持居中。
- 标题横条：居中放置，半透明白色背景，`backdrop-filter: blur(...) saturate(...)`，圆角、细白描边、顶部高光和轻微阴影。
- 标题：默认白色、粗字重、居中，允许调节字号和颜色。
- 副标题：较小字号、较低不透明度，默认居中。
- 控件：灰蓝色和暖白色为主，不使用大面积紫色渐变。

## 推荐实现方案

使用 DOM 实时预览 + Canvas 导出 PNG。

- DOM 负责交互和预览：CSS 原生实现响应式布局、圆角和毛玻璃效果，保证输入和字体切换即时反馈。
- Canvas 负责导出：按同一份编辑状态重新绘制背景图、玻璃横条和文字，输出 1600 × 900 的 PNG。
- 不使用 HTML 截图依赖，避免额外运行时依赖和离线加载问题。
- 使用 Vite + React + TypeScript + 原生 CSS。运行时只依赖 React 及构建工具，不从网络请求素材。

## 页面结构

### `EditorShell`

承载页面布局、顶部栏和当前编辑状态。桌面端为左侧控制面板、右侧画布；窄屏时改为上方画布、下方控制面板。

顶部栏包含：

- 产品名称 `Cover Atelier`。
- 当前画布比例提示 `16:9`。
- 导出 PNG 按钮。

### `ControlPanel`

按使用顺序组织为四组：

1. 背景：本地图片选择器、重新选择入口、当前文件名。
2. 文字：标题输入框、副标题输入框。
3. 字体：预设字体选择、本地字体文件上传、字号、文字颜色。
4. 玻璃横条：横条宽度、横条垂直位置、透明度、模糊强度、圆角半径。

默认值：

```ts
type CoverState = {
  backgroundUrl: string | null;
  backgroundName: string | null;
  title: string;
  subtitle: string;
  fontFamily: string;
  fontSize: number;
  textColor: string;
  stripWidth: number;
  stripPositionY: number;
  stripOpacity: number;
  blurAmount: number;
  stripRadius: number;
};

const defaultCoverState: CoverState = {
  backgroundUrl: null,
  backgroundName: null,
  title: "写下你的标题",
  subtitle: "Add a subtitle or tagline",
  fontFamily: "system-ui",
  fontSize: 52,
  textColor: "#ffffff",
  stripWidth: 78,
  stripPositionY: 50,
  stripOpacity: 0.3,
  blurAmount: 24,
  stripRadius: 24,
};
```

### `CoverCanvas`

将 `CoverState` 映射为预览画面：

- 背景图以绝对定位元素铺满画布。
- 没有图片时显示内置 SVG/CSS 山景占位背景，保证初次打开就有可编辑画面。
- 玻璃横条根据百分比宽度和垂直位置定位。
- 标题和副标题共用横条内部的居中排版容器。
- 画布使用 `aspect-ratio: 16 / 9`，并设置最大宽度避免在大屏上失去编辑焦点。

### `ExportRenderer`

导出时等待 `document.fonts.ready`，再将当前状态绘制到离屏 Canvas：

1. 创建 1600 × 900 Canvas。
2. 计算背景图 cover 裁剪区域，绘制到完整画布。
3. 对横条区域复制背景图像素，使用 Canvas blur filter 绘制模糊层，并叠加半透明白色填充、边框和圆角裁剪。
4. 使用当前字体、字号、颜色和文本内容绘制标题与副标题。
5. 调用 `canvas.toBlob()` 创建 PNG，并通过临时下载链接保存为 `cover-atelier-YYYYMMDD-HHmm.png`。

导出实现允许预览和 PNG 在视觉上存在极小的浏览器差异，但背景裁剪、横条位置、文字内容和主要视觉效果必须保持一致。

## 字体策略

首版不依赖网络字体。字体选择包含本机安全字体栈，例如系统无衬线、系统衬线、楷体风格和等宽字体。用户还可以选择 `.ttf`、`.otf` 或 `.woff` 文件：

- 使用 `FileReader` 读取本地文件。
- 使用 `FontFace` 创建内存字体并调用 `document.fonts.add()`。
- 字体只保存在当前页面会话中，不上传、不写入服务器。
- 加载失败时显示提示并回退到系统字体。

## 本地数据与隐私边界

- 图片、字体和编辑状态只存在当前浏览器标签页内存中。
- 不调用外部 URL，不写入数据库，不建立账号。
- 导出文件由浏览器下载到用户选择的本地下载目录。
- 页面刷新后默认恢复初始状态；首版不承诺持久化编辑内容。

## 异常处理

- 文件选择为空：保持当前画面，不清空已有背景。
- 图片格式无法读取：显示“无法读取图片，请选择 PNG、JPG 或 WebP”，保留当前编辑状态。
- 字体加载失败：显示失败提示，继续使用上一套可用字体。
- 浏览器不支持 `backdrop-filter`：使用不透明度更高的半透明白色背景作为降级。
- Canvas 导出失败：显示“导出失败，请重试”，不重置编辑状态。

## 响应式行为

- 桌面端：控制面板固定在左侧，画布位于右侧并自适应剩余空间。
- 平板端：缩小控制面板宽度，画布保持 16:9。
- 移动端：画布置顶，控制组改为纵向排列，导出按钮固定在顶部栏可见区域。
- 不使用 `window` 滚动监听；布局通过 CSS 媒体查询完成。

## 验收与测试

功能测试：

- 初次加载时可以看到占位封面和默认文字。
- 选择本地图片后，画布更新且没有网络请求。
- 修改标题、副标题、字体和玻璃参数时，预览即时更新。
- 上传可用字体后，预览能显示该字体；导出前等待字体加载完成。
- 点击导出可以生成可打开的 1600 × 900 PNG。
- 传入无效图片或字体时，页面不崩溃，编辑状态不丢失。

响应式测试：

- 桌面宽屏、平板宽度和手机宽度下布局均可操作。
- 开启减少动画设置时，页面不依赖动画才能完成编辑或导出。

构建测试：

- `npm run build` 成功退出。
- 构建产物不包含远程资源依赖。
- 使用本地静态服务可在断网状态下打开并完成编辑、导出。
