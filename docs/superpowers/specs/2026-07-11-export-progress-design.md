# 导出进度浮窗设计

## 背景

当前导出流程由 `EditorShell` 调用异步 `exportCover` 完成。PNG、JPEG 和 WebP 通常很快完成，但 AVIF 需要加载 WASM 编码器并进行质量优先编码，用户在等待期间缺少反馈。

## 方案

采用方案 A：居中玻璃浮窗。

导出开始后，在页面上方显示半透明遮罩和居中的玻璃卡片。卡片显示“正在导出封面”、目标文件名、当前导出格式、阶段文字、百分比和横向进度条。浮窗不提供取消按钮，避免导出过程中中断画布编码；导出按钮和编辑控件在导出期间保持禁用状态。

## 数据流

`EditorShell` 持有导出进度状态，并将进度回调传给 `exportCover`。`exportCover` 在以下阶段报告进度：

1. 准备画布：0–10%
2. 加载背景图片：10–25%（无背景图时跳过）
3. 绘制背景、玻璃横条和文字：25–58%
4. 编码当前格式：58–92%
5. 创建下载文件：92–100%

进度是阶段性反馈，不伪造编码器内部的连续进度。AVIF 编码期间维持在编码阶段并配合轻微动画，明确告诉用户正在进行质量优先编码。

## 组件与可访问性

新增独立的 `ExportProgressModal` 组件，负责浮窗结构和视觉呈现；导出逻辑仍由 `exportCover` 负责。浮窗使用 `role="dialog"`、`aria-modal="true"` 和明确的标题；进度条使用 `role="progressbar"`、`aria-valuemin`、`aria-valuemax` 和 `aria-valuenow`。导出完成或失败后浮窗自动关闭，失败信息继续由现有状态提示区域展示。

## 视觉

沿用现有深色工作台和玻璃拟态语言：页面遮罩、深色半透明卡片、珊瑚色进度填充、圆角边框和微弱阴影。移动端浮窗宽度使用页面内边距约束，长文件名截断显示但保留完整可访问名称。

## 测试

- 单元测试验证导出阶段按顺序报告且进度单调递增。
- 组件静态渲染测试验证浮窗包含文件名、阶段文字、进度百分比和无障碍属性。
- 运行完整 Vitest 测试、生产构建和 GitHub Pages 构建。
- 使用浏览器触发一次 AVIF 导出，确认浮窗出现、进度更新、导出完成后关闭。
